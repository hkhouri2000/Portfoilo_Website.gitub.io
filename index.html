    Tracking Objects in Videos
      Project Information

      Project Information

      The project involves implementing object tracking algorithms, specifically the Lucas-Kanade tracker and the Matthew-Baker (inverse compositional) method. The goal is to track an object in a video by updating an affine transform that warps each frame to align with a template defined in the first frame.

      In the Lucas-Kanade tracker, the algorithm minimizes the sum of squared differences between the warped image and the template. It uses a linear approximation to find the optimal alignment. The Jacobian and Hessian matrices are computed iteratively to update the parameters of the warp.

      The Matthew-Baker method is a more computationally efficient version of the Lucas-Kanade tracker. It utilizes the inverse compositional approach, where the warp is inverted and composed with the template image. This reduces the computational complexity compared to the forward additive alignment used in Lucas-Kanade.

      The project provides two video sequences: a car on a road and a helicopter approaching a runway. The initialization step involves drawing a bounding box around the object to be tracked in the first frame. The tracker then updates the warp parameters for each subsequent frame, aligning the template with the warped current frame.

      The computational complexity of the initialization step and each runtime iteration of the Inverse Compositional method needs to be determined and compared to the regular Lucas-Kanade method. The complexity should be expressed in terms of the number of pixels in the template and input image, as well as the number of parameters used for the warp.

      Overall, the project focuses on implementing and understanding object tracking algorithms, applying them to provided video sequences, and analyzing their computational efficiency.

    (3D point to 2D) Point and Inverse (2D point to 3D ray) Camera Projection
      Project Information

      Project Information

      Overview

      This project focuses on implementing forward and inverse camera projection in the context of computer vision. The goal is to project 3D coordinates of body joints onto 2D image pixel coordinates and then reconstruct the original 3D coordinates using triangulation from two camera views. The project involves understanding the pinhole camera model, camera calibration parameters, and the transformation chain from 3D to 2D and back again.

      Tasks

      1. Parsing and interpreting the 3D joint data from motion capture recordings.
      2. Reading and understanding the camera calibration parameters for two video cameras.
      3. Projecting 3D joint data into pixel coordinates in each camera view using the camera parameters.
      4. Reconstructing the 3D location of each joint from the projected 2D points using triangulation.
      5. Computing the Euclidean distance between the original and reconstructed 3D joints to measure reconstruction error.
      6. Exploring epipolar geometry by computing and displaying epipolar lines between the two camera views.

      Skills and Tools

      The project requires Matlab programming skills and provides an opportunity to gain practical experience in computer vision, including intrinsic and extrinsic camera parameters, projection of 3D points into 2D images, triangulation for 3D reconstruction, and evaluating the accuracy of the reconstructed points. The quantitative evaluation involves calculating L2 distances between the original and reconstructed points and analyzing statistics such as mean, standard deviation, minimum, median, and maximum distances.

      Conclusion

      Overall, the project aims to deepen understanding and implementation of camera projection and triangulation techniques in computer vision, while also providing insights into the challenges and considerations involved in practical applications of these methods.

    The Image Processing Pipeline

      Project Information

      In this project, I developed a basic image processing pipeline using Matlab. The purpose of the project was to familiarize myself with Matlab as a tool for manipulating images. The project involved several tasks, which are summarized below:

      Initials

      I loaded the image into Matlab and determined its properties, including the number of bits per integer, width, and height. I then converted the image into a double-precision array.

      Linearization

      The image's 2D-array was not a linear function of the true energy received by each pixel. I applied a linear transform to the image, mapping specific pixel values to a range of [0, 1]. I also clipped negative values to 0 and values greater than 1 to 1.

      Identifying the correct Bayer pattern

      Most cameras capture images using a Bayer pattern, where each pixel captures only one color channel. I developed a method to identify the Bayer pattern used in the image, which could be one of the four possible patterns: 'grbg', 'rggb', 'bggr', or 'gbrg'.

      White balancing

      Cameras apply white balancing to adjust the relative weights of the red, green, and blue measurements, ensuring that white objects appear white in the image. I implemented both gray world and white world white balancing algorithms and compared the results.

      Demosaicing

      After white balancing, I performed demosaicing to convert the partial color channels into three full-resolution color channels. I used bilinear interpolation for demosaicking, utilizing Matlab's interp2 function.

      Brightness adjustment and gamma correction

      I adjusted the image's brightness by linearly scaling it. The scaling factor was selected as a percentage of the pre-brightening maximum grayscale value. I also applied a non-linear gamma correction transform to match the sensor measurements to the sRGB standard.

      Compression

      Finally, I stored the processed image in two formats: PNG (no compression) and JPEG (with quality setting 95). I compared the compressed and uncompressed images, analyzed the compression ratio, and determined the lowest JPEG quality setting that resulted in indistinguishable image quality.

      The project allowed me to gain hands-on experience with image processing techniques and the use of Matlab for manipulating and analyzing images. It enhanced my understanding of the image processing pipeline and provided an opportunity to apply various transformations and algorithms to improve image quality and visual appearance.

    Field-Programming Gate Array (FPGA) MIPS Five Stage Pipelined CPU

      Project Information

      In this project, I implemented a pipelined MIPS processor datapath with various features. The project consisted of two main parts:

      Part 1

      I implemented a 5-stage pipeline (FDEMW) with branch resolution in the Decode (D) stage and non-memory hazard detection in the Decode (D) stage. Additionally, a 2-cycle pipelined multiplication unit spanning the Execute (E) and Memory (M) stages was included. For Part 1, I was responsible for implementing hazard detection logic for all hazards and forwarding logic for the EX stage for some hazards, as well as forwarding logic for the M stage for all hazards.

      Part 2

      I implemented branch prediction extensions, including a 2K-entry bimodal table of 2-bit predictors with an initial state of "10." Additionally, a 32-entry Branch Target Buffer (BTB) was implemented to cache the contents (not the address) of supported branch and jump instructions (BEQ, BNE, J, JAL). Instructions like JR and JALR were explicitly not cached due to target indirection.

      For both parts, I implemented an instruction cache and a data cache as replacements for the Instruction Memory (IMEM) and Data Memory (DMEM), respectively, in the memory hierarchy. The instruction cache, a 2KB direct-mapped cache with a block size of 64 bytes, served cache misses from the IMEM. The data cache, a 2KB 2-way associative cache with an inverse-MRU eviction policy, was used to serve cache misses from the DMEM. The DMEM had a write-back policy and stall-allocated on store misses. Evicted dirty lines were written back to the DMEM.

      Throughout the project, I worked with a provided framework that included files for hazard detection, forwarding logic, the branch history table (BHT), and the implementation of the caches. The project was divided into milestones, and for Milestone 1, I completed parts 1 and 2, utilizing a dummy memory hierarchy to test the design before integrating the caches.

      The framework files relevant to my edits and additions for parts 1 and 2 were:

      • HazardDetection.v: Implementation of hazard detection logic.
      • EXForwarding.v: Implementation of forwarding logic for the Execute (EX) stage.
      • MEMforwarding.v: Implementation of forwarding logic for the Memory (MEM) stage.
      • BHT.v: Implementation of the branch history table for branch prediction.

      Overall, this project allowed me to gain experience in designing and implementing a pipelined MIPS processor datapath with various pipeline stages, hazard detection, forwarding logic, branch prediction, and memory hierarchy using instruction and data caches.
